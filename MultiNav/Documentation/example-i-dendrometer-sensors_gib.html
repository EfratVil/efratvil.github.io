MultiNav - Multivariate Explanatory Data Analysis

Chapter 8 Example I - Dendrometer Sensors

The first dataset corresponds to the challenge of identifying malfunctioning dendrometer sensors. Dendrometers are measurement devices that can continuously monitor and record size variation in plant organs like stems, branches and fruits. Data from dendrometers can be used for monitoring of plants’ daily water status, and accordingly make irrigation decisions.The challange is to identify the malfunctioning sensors with multivariate anomaly detection methods and filtering them out, before making irrigation decisions.


Loading:

library(MultiNav)
library(MASS)
library(robustbase)
library(huge)
library(corrplot)
library(data.table)

8.1 Exploratory Data Anaylsis with First Order Scores

We start by loading the DendrometerSensors dataset and perform initial exploration to get a feel of how the data looks and to check if there are any sensors that have obvious anomalies. We do this by examining the data with the default first order scores. The dendrometer dataset includes data of 100 sensors from 120 hours readings (5 days). The data is already pre-processed to ensure distributions are symmetric.

data(DendrometerSensors)
data <- DendrometerSensors

# We will examin anomalies in a window of last 72 hours.
mat<-data[49:120,]

MultiNav(mat,type = "snapshot scores")



Hoovering over a point will show the corresponding sensor reading layed on top of the functional boxplot background. Several anomalies can be easily identified by the first order scores (104, 142, 161, 182).

8.2 Evaluate Multivariate Scoring methods

Next several multivariate anomaly detection scores will be calculated. At the first evaluation phase, it may not be clear yet which methods are most suitable for this spesific dataset. We will evaluate and compare scores with the “snapshot scores” display will be used.

We will remove hours where the standard diviation equals 0. Then calculate the anomaly scores and store them in a data.table called Calculated.Scores. Since we use robust multivarite anomaly scores, it is not naccesary to exclude the sensors that were already identified as anomalies.

#exclude.sensors <- match(c("104","142", "161","182"), names(mat))
#mat<-mat[,-exclude.sensors]
exclude.hours<- c(1,24,25,48,49,72)
mat<-mat[-exclude.hours,]

T2 <- T2(t(mat))
T2_mcd <- T2(t(mat), cov="mcd")
T2_co <- round(T2(t(mat), cov="Croux-Ollerer"),5)
EigenvectorCentrality<-EigenCentrality(mat)


Calculated.Scores <- as.data.table(cbind(id=as.numeric(colnames(mat)), T2=T2,T2_mcd=T2_mcd, T2_co=T2_co, EigenvectorCentrality))
head(Calculated.Scores)
